# 测试与质量保证技能 (Testing & QA Skill)

本技能旨在规范测试流程，确保前端和后端代码的质量、功能的正确性以及系统的稳定性。

## 核心测试流程

开发工作遵循“后端 -> 前端 -> 联调”的顺序，每个阶段都包含必要的测试环节。

1.  **后端单元/集成测试**:
    - 在完成后端功能开发后，**必须**对 `Service` 层和 `Controller` 层编写测试用例。
    - 使用 Mockito 等框架模拟依赖（如 `DAO` 层或其他服务），隔离测试单元。
    - **必须**保留必要的测试用例。

2.  **后端接口测试**:
    - 使用 Postman 或其他 API 测试工具，对后端暴露的接口进行完整测试。
    - 验证所有正常场景和异常场景（如参数错误、权限不足等）。
    - 确保接口返回的数据结构符合 `{"returnCode": "SUC0000", "body": "{...}"}` 规范。

3.  **前端验证**:
    - 在完成前端页面开发后，**必须**通过以下强制验证流程：
        1. `npm run typecheck`
        2. `npm run build`
        3. `npm run lint -- --fix`

4.  **前后端联调测试**:
    - 在前后端开发都完成后，启动前后端服务，进行端到端的联调测试。
    - 重点验证数据流、用户交互和权限控制是否符合预期。
    - 确保在真实交互场景下，`RequireAuth` 等路由守卫正常工作。

## 关键测试要点

- **更新/删除操作测试**:
  - 测试用例**必须**验证 `UPDATE` 和 `DELETE` 操作是基于**唯一标识符 (ID)** 完成的。
  - **必须**包含测试场景：当尝试使用非唯一字段（如 `name`）进行更新/删除时，操作应失败或被拒绝。

- **权限测试**:
  - **必须**测试不同角色的用户访问权限。
  - 验证未授权用户无法访问受保护的页面和 API。
  - 验证菜单权限控制是否生效，没有权限的菜单不应显示。

- **边界与异常测试**:
  - 对输入框、表单等进行边界值测试（如最大/最小长度、特殊字符等）。
  - 模拟 API 请求失败或返回错误码，验证前端是否能优雅地处理并给出提示。

- **登入/登出流程测试**:
  - 验证用户登出后，`localStorage` 中的 `token` 和用户信息被完全清除。
  - 验证已登出的用户无法通过直接访问 URL 进入内部页面，应被重定向到登录页。
