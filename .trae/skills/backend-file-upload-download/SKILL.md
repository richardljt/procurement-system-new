# 后端文件上传下载规范

本规范旨在统一后端的文件处理方式，确保代码的一致性、可维护性和用户体验。

## 何时调用此技能

- 在开发任何需要**文件上传**的后端接口时。
- 在开发任何需要**文件下载**的后端接口时。
- 在设计或修改与**附件、图片、文档**等文件存储相关的业务逻辑时。

## 核心架构

- **统一接口**: 所有文件操作必须通过统一的 `FileStorageService` 接口进行，包括文件保存和获取
- **两种实现**：一种实现是直接将文件保存到本地文件系统，另一种实现是将文件保存到 Amazon S3，可以通过配置切换实现方式。默认采用本地文件系统实现，工程所在目录/upload中。
- **文件Key**: 使用 `/{module}/{generated_snowflake_id}.{ext}` 格式生成唯一的对象存储 Key，同时保留原文件名在数据库中，供显示或下载时还原
    - `{module}`: 业务模块的全局唯一名称 (例如: `presale`)。
    - `{snowflake_id}`: 雪花算法生成的唯一 ID。
    - `.{ext}`: 文件原始扩展名。
- **本地缓存**: 为了提升性能和用户体验，采用**本地文件系统**作为二级缓存。
    - **缓存路径**: 缓存路径配置文件中指定，默认值为:`[工程所在目录]/upload` 中

## 文件上传流程

1.  **接收文件**: 控制器接收到前端传来的文件。
2.  **生成并存储元数据**: 参考文件Key的要求生成文件唯一标识
    - **文件Key**: 使用 `/{module}/{snowflake_id}.{ext}` 格式生成唯一的对象存储 Key。
        - `{module}`: 业务模块的全局唯一名称 (例如: `presale`)。
        - `{snowflake_id}`: 雪花算法生成的唯一 ID。
        - `.{ext}`: 文件原始扩展名。
3.  **存储文件**：使用后端容器注入默认的`FileStorageService`实现类，调用其`saveFile`方法存储文件。
4   **数据库记录**: 在**统一的文件信息表** (`file_record`) 中，记录文件的元数据，包括 `module`, `original_file_name`, `file_key` (即 S3 Key), `file_size` 等。
5.  **返回文件Key**: 统一的文件上传接口在完成本地缓存后，应立即向前端返回**文件Key** (`file_key`)，形如：`/presale/123456789012345678.pdf`
6.  **业务表关联**: 调用方（业务 Service）获取到文件 Key 后，将其与业务实体的保存过程合并，保存在**对应业务实体表的字段**中
    - **业务表多字段**：业务实体如果有多类型的文件列表应区分保存，比如：采购申请中有产品相关文件`product_files`和单一来源采购说明文件`attachment_files`字段，要在业务表中不同字段区分保存
    - **多文件**: 如果一个字段需要关联多个文件，应使用**逗号 (,)** 分隔多个文件 ID (或 Key)。

## 文件下载流程

1.  **接收 Key**: 统一的文件下载接口接收前端传来的**文件 Key**。
2.  **查询文件信息**： 根据文件 Key 查询**统一的文件信息表** (`file_record`)，获取文件的元数据，包括 `module`, `original_file_name`, `file_key`, `file_size` 等。
3.  **查询文件本身**： 使用后端容器注入默认的`FileStorageService`实现类，调用其`readFile`方法读取文件，获得文件对象
4.  **返回文件**： 将查询到的文件信息对象返回给前端，前端在展示或下载时，注意用户看到的时`original_file_name`，而不是`file_key`。

## 示例：售前模块上传产品信息（用本地文件实现）

1.  用户在“售前”模块上传 `产品信息.pdf`。
2.  后端接收文件，自定义模块名为 `presale`。
3.  生成雪花 ID，例如 `123456789012345678`。
4.  生成 S3 Key: `/presale/123456789012345678.pdf`。
5.  文件保存到 `/upload/presale/123456789012345678.pdf`。
6.  在 `file_record` 表中插入一条记录，包含文件名、Key 等信息。
7.  上传接口返回 Key `/presale/123456789012345678.pdf`。
8.  售前业务逻辑将此 Key 保存到对应的业务表字段中。
