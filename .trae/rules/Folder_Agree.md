# 工程目录要求

为了保持项目结构的清晰和一致性，所有新功能和模块的开发都应遵循以下目录规范。

## **顶层结构**

- `/frontend`: 存放所有前端相关的代码、资源和配置文件。
- `/backend`: 存放所有后端相关的代码、资源和配置文件。
- `/docs`: 存放项目相关的各类文档，例如需求文档（'/PRD')、技术文档、学习记录等。

## **详细说明**

### `/frontend`

该目录是前端工程的根目录，包含了所有与用户界面、交互逻辑相关的代码。

- **`src/`**: 前端应用的核心源代码。
- **`public/`**: 静态资源，不会被构建工具处理。
- **`dist/`**: `npm run build` 生成的构建产物，用于生产环境部署。
- **`package.json`**: 项目依赖和脚本配置。
- **`vite.config.ts`**: Vite 配置文件。
- **`tailwind.config.js`**: Tailwind CSS 配置文件。

### `/backend`

该目录是后端工程的根目录，包含了所有业务逻辑、数据处理和 API 服务。

- **`src/`**: 后端应用的核心源代码 (Java)。
- **`target/`**: `mvn package` 生成的构建产物，包含可执行的 `.jar` 文件。
- **`pom.xml`**: Maven 项目配置文件。

这样的结构可以确保前后端代码的完全分离，便于独立开发、构建和部署。
