# 详细需求

## 基础框架搭建

参考 [采购申请页面.html](./采购申请页面.html), [采购列表页面.html](./采购列表页面.html), [采购流程页面.html](./采购流程页面.html) 设计中的公共部分完成基础框架搭建。包含：
- 基础的用户认证部分
- react的全局处理部分
- header组件的抽取，包含菜单、通知、配置和登陆用户头像等
- 左侧菜单组件抽取，包含菜单和权限配置，不同用户有不同的菜单权限

## 采购申请页面搭建
参考 [采购申请页面.html](./采购申请页面.html) 完成，有如下验收标准：
- 用户选择使用系统全局接口，对接到第三方系统获取用户信息，可以先实现mock接口
- 添加事前申请一样，要先定义interface接口，率先在本地实现，插入一些示例信息
- 供应商信息一样，要先定义interface接口，率先在本地实现，插入一些示例信息
- 提交时，页面所有信息都应该按结构持久化到数据库，并提示给用户创建成功的信息，如果创建失败要给予用户业务型提示
- 提交成功后，返回到 [采购列表页面.html](./采购列表页面.html)

## 采购列表页面搭建
参考 [采购列表页面.html](./采购列表页面.html) 完成，有如下验收标准：
- 页面上显示的状态信息都应该能够在后端数据库查询到
- 样式要注意统一

## 采购流程页面搭建
参考 [采购流程页面.html](./采购流程页面.html)  完成，有如下验收标准：
- 流程页面中的部分元素会超出采购申请页面的元素，是要在别的功能维护，你要优先按业务拆分到不同数据库表，并识别页面上的元素分属到不同的表中
